# Backend 도메인 규칙

## ENUM 정의

### ArcanaType (아르카나 구분)

| 값 | 설명 |
|---|---|
| MAJOR | 메이저 아르카나 (22장) |
| MINOR | 마이너 아르카나 (56장) |

### Suit (수트)

| 값 | 설명 | 해당 카드 |
|---|---|---|
| WANDS | 완드 | 마이너 아르카나 |
| CUPS | 컵 | 마이너 아르카나 |
| SWORDS | 소드 | 마이너 아르카나 |
| PENTACLES | 펜타클 | 마이너 아르카나 |
| NONE | 없음 | 메이저 아르카나 |

### ConsultationStatus (상담 상태)

| 값 | 설명 | 전이 가능 상태 |
|---|---|---|
| PENDING | AI 해석 요청 중 | → COMPLETED, FAILED |
| COMPLETED | 해석 완료 | 없음 (최종) |
| FAILED | 해석 실패 | 없음 (최종) |

상태 전이도
PENDING ──→ COMPLETED
└──→ FAILED

### CardPosition (카드 포지션)

| 값 | 설명 |
|---|---|
| 1 | 과거 (현재 상황의 원인) |
| 2 | 현재 (지금 처한 상황) |
| 3 | 미래 (앞으로의 흐름) |

## 비즈니스 규칙

### 회원 (users)

| ID | 규칙 | 적용 위치 |
|---|---|---|
| U-001 | 이메일은 시스템 내 유일해야 한다 | DB UNIQUE + 서비스 레이어 |
| U-002 | 비밀번호는 BCrypt로 암호화하여 저장한다 | 서비스 레이어 |
| U-003 | 탈퇴한 회원은 deleted_at이 NULL이 아니다 | DB + 서비스 레이어 |
| U-004 | 탈퇴한 회원의 이메일로는 재가입이 불가하다 | 서비스 레이어 |

### 타로 카드 (tarot_cards)

| ID | 규칙 | 적용 위치 |
|---|---|---|
| C-001 | 카드는 총 78장이며 card_number는 0~77 이다 | DB UNIQUE + CHECK |
| C-002 | 메이저 아르카나(22장)의 suit는 NONE이다 | DB CHECK + 서비스 레이어 |
| C-003 | 카드 마스터 데이터는 생성 후 수정하지 않는다 | 서비스 레이어 (수정 API 미제공) |

### 상담 (consultations)

| ID | 규칙 | 적용 위치 |
|---|---|---|
| CO-001 | 비로그인 사용자는 상담을 생성할 수 없다 | Spring Security |
| CO-002 | 고민 내용은 최소 10자, 최대 500자이다 | 서비스 레이어 + 프론트 유효성 검증 |
| CO-003 | 상담 생성 시 초기 상태는 PENDING이다 | 서비스 레이어 |
| CO-004 | COMPLETED / FAILED 상태는 되돌릴 수 없다 | 서비스 레이어 |
| CO-005 | 본인의 상담 기록만 조회할 수 있다 | 서비스 레이어 (403 처리) |
| CO-006 | 상담 삭제는 소프트 딜리트로 처리한다 | 서비스 레이어 |
| CO-007 | 삭제된 상담은 목록/상세 조회에서 노출되지 않는다 | QueryDSL WHERE deleted_at IS NULL |

### 상담 카드 (consultation_cards)

| ID | 규칙 | 적용 위치 |
|---|---|---|
| CC-001 | 상담당 카드는 반드시 3장이어야 한다 | 서비스 레이어 |
| CC-002 | 동일 상담에서 같은 포지션은 중복될 수 없다 | DB UNIQUE (consultations_id, position) |
| CC-003 | 동일 상담에서 같은 카드를 중복 선택할 수 없다 | 서비스 레이어 |
| CC-004 | 카드 포지션은 생성 후 변경할 수 없다 | JPA updatable = false |
| CC-005 | 포지션은 1(과거), 2(현재), 3(미래) 만 허용한다 | DB CHECK + 서비스 레이어 |

## 유효성 검증 기준

### 회원가입

| 필드 | 조건 | 오류 메시지 |
|---|---|---|
| email | 이메일 형식, 필수 | "올바른 이메일 형식을 입력해주세요" |
| email | 중복 불가 | "이미 사용 중인 이메일입니다" |
| password | 최소 8자, 영문+숫자 조합, 필수 | "비밀번호는 영문, 숫자를 포함한 8자 이상이어야 합니다" |
| nickname | 최소 2자, 최대 20자, 필수 | "닉네임은 2자 이상 20자 이하여야 합니다" |

### 상담 등록

| 필드 | 조건 | 오류 메시지 |
|---|---|---|
| concern | 최소 10자, 최대 500자, 필수 | "고민 내용은 10자 이상 500자 이하로 입력해주세요" |
| cards | 반드시 3장 선택 | "타로 카드를 3장 선택해주세요" |
| cards | 중복 카드 불가 | "같은 카드를 중복 선택할 수 없습니다" |
