# 화면 설계 — Pages & Routing

## URL 구조

| URL | 컴포넌트 | 인증 필요 | 설명 |
|---|---|---|---|
| `/login` | `LoginPage` | ✗ | 로그인 |
| `/signup` | `SignupPage` | ✗ | 회원가입 |
| `/` | `HomePage` | ✓ | 메인 홈 (타로 상담 시작) |
| `/cards` | `CardSelectPage` | ✓ | 카드 3장 선택 |
| `/consultation/:id` | `ConsultationPage` | ✓ | AI 해석 결과 |
| `/history` | `HistoryPage` | ✓ | 상담 기록 목록 |
| `*` | redirect → `/` | — | 없는 경로 처리 |

- 인증 필요 라우트는 `PrivateRoute`로 감싸 토큰 없으면 `/login` 리다이렉트
- 로그인 상태에서 `/login`, `/signup` 접근 시 `/`로 리다이렉트

## 페이지 플로우

```
[비로그인]
  ┌──────────────┐
  │  /login      │ ←──────────────────────────────┐
  │  로그인       │                                │
  └──────┬───────┘                                │
         │ 성공                    회원가입 링크    │
         │          ┌──────────────┐              │
         │          │  /signup     │              │
         │          │  회원가입     │──────────────┘
         │          └──────┬───────┘
         │                 │ 성공 → /login 리다이렉트
         ▼
[로그인 상태]
  ┌──────────────────────────────────┐
  │  /  (HomePage)                   │
  │  · 고민 입력 인풋                 │
  │  · "타로 상담 시작" 버튼          │
  │  · 최근 상담 기록 (최대 3개 미리보기)│
  └──────┬───────────────────────────┘
         │ 고민 입력 + 상담 시작 버튼
         ▼
  ┌──────────────────────────────────┐
  │  /cards  (CardSelectPage)        │
  │  · 고민 텍스트 표시 (상단)        │
  │  · 78장 카드 캐러셀 (셔플 후)     │
  │  · 3장 순서대로 선택              │
  │    - 카드1: 과거                 │
  │    - 카드2: 현재                 │
  │    - 카드3: 미래                 │
  │  · "해석 받기" CTA              │
  └──────┬───────────────────────────┘
         │ 카드 3장 확정 → POST /consultations
         ▼
  ┌──────────────────────────────────┐
  │  /consultation/:id               │
  │  (ConsultationPage)              │
  │  · AI 해석 로딩 애니메이션        │
  │  · 채팅 버블 형식 해석 표시       │
  │  · 선택한 카드 3장 요약 칩        │
  │  · 바텀시트: 카드별 상세 해석     │
  │  · "다시 상담하기" 버튼          │
  │  · FAILED 시: "새 상담 시작" 버튼 (→ /) │
  └──────┬───────────────────────────┘
         │ 헤더 히스토리 아이콘
         ▼
  ┌──────────────────────────────────┐
  │  /history  (HistoryPage)         │
  │  · 상담 기록 카드 목록 (날짜순)   │
  │  · 각 항목: 날짜 / 카드 3장 썸네일│
  │  · 탭: 기록 없을 때 빈 상태 UI   │
  │  · 스와이프 → 삭제 (소프트 딜리트)│
  └──────────────────────────────────┘
```

## 화면별 상태 정의

### LoginPage / SignupPage
| 상태 | 설명 |
|---|---|
| idle | 초기 폼 |
| loading | 버튼 비활성화 + 텍스트 변경 |
| error | 인라인 에러 메시지 표시 |

### CardSelectPage
| 상태 | 설명 |
|---|---|
| idle | 78장 뒤집힌 카드 캐러셀 |
| selecting | 카드 선택 중 (1~3장) — 선택 수 표시 |
| ready | 3장 선택 완료 — CTA 활성화 |
| submitting | API 요청 중 로딩 |

### ConsultationPage
| 상태 | 설명 |
|---|---|
| loading | AI 해석 대기 (타이핑 애니메이션) |
| done | 해석 완료 + 인터랙션 활성화 |
| error | "새 상담 시작" 버튼 노출 → 홈(/)으로 이동 |

### HistoryPage
| 상태 | 설명 |
|---|---|
| loading | 스켈레톤 UI (초기 로드) |
| empty | 빈 상태 일러스트 + CTA |
| filled | 기록 카드 목록 (무한 스크롤) |
| loadingMore | 추가 페이지 로드 중 (하단 스피너) |

**무한 스크롤 동작**
- 초기 로드: page=0, size=10
- 스크롤이 하단 100px 이내에 닿으면 다음 페이지 요청
- `hasNext: false` 응답 시 더 이상 요청하지 않음
- TanStack Query `useInfiniteQuery` 사용
